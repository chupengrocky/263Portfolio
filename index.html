USER EVALUATION

USER EVALUATION
Issue Evidence Solution Priority
Cannot filter product by conditioning the item page All participants are interested in clicking the Filter button but the Filter button does not redirect to any other pages or popped windows. Need to implement filter feature 2
Users cannot see what they have order User would like to see a page containing the item they have already bought Need order status in profile 1
Community page’s name does not clearly show its functionality Participants think the Community Page is a group chat or a “community” for people to know each other. Actually we plan to let user see whether there are seller or buyer nearby Change the community page to “Nearby page” 1
Currently there is no actual price listing for the items so customers can’t tell how much it costs Users were confused at a blatant lack of an explicit price tag Need to add price to items 1
Users may not know how to return from the confirmation page After participants finally buy items from the app, they can not find the button to go back to the home page. They are stuck in the checkout page. Add an obvious return home button to confirmation page 2
In the report page, there is no place to upload a picture Participants want to add pictures when they report an item but they can't find a place to upload pictures. Add upload pic in the addition info section 2
Back button in chat box page is kind misleading When participants click the back button on the chat box page they seem to expect to go back to the item page rather than the chat list page. Chat box should have a button to return the item detail page 1
There is no way to see at a glance what condition the items are in aside from photos, which can be small and hard to see Users suggested having a dedicated condition section as part of the item page Add condition of item to item details 3
